Introduction

欢迎使用 eSIM Partner API。本文档提供了完整的 API 接口说明，帮助您快速集成 eSIM 服务。

Base URL

https://api.esim-platform.com/api/v1/open

认证方式

所有 API 请求需要在 Header 中携带以下认证信息：

Header 说明
X-Access-Code 您的 AccessCode
X-Signature 请求签名（HMAC-SHA256）
X-Timestamp 请求时间戳（毫秒）

Partner API

Partner API 提供了完整的 eSIM 套餐查询、订购、管理等功能。

Get All Data Packages

获取所有可用的数据套餐列表。

POST /packages/list

Request Parameters

参数 类型 必填 说明
region string 否 地区筛选，如 "ASIA", "EU"
country string 否 国家代码，如 "JP", "KR"

Response Example

{
  "code": 0,
  "message": "success",
  "data": {
    "packages": [
      {
        "packageCode": "pkg_1001",
        "name": "亚洲 7 天 1GB",
        "type": "REGIONAL",
        "region": "ASIA",
        "dataCapacity": "1GB",
        "validity": 7,
        "price": 18.00
      }
    ]
  }
}

Order Profiles

订购 eSIM Profile，支持批量订购。

POST /profiles/order

Request Parameters

参数 类型 必填 说明
packageCode string 是 套餐代码
quantity integer 是 订购数量（1-100）
merchantOrderNo string 是 商户订单号（唯一）

Response Example

{
  "code": 0,
  "message": "success",
  "data": {
    "orderNo": "ORD-20250227001",
    "profiles": [
      {
        "iccid": "89852342022273174242",
        "activationCode": "LPA:1$smdp.example.com$...",
        "qrCode": "data:image/png;base64,..."
      }
    ]
  }
}

Query All Allocated Profiles

查询所有已分配的 Profile 列表。

POST /profiles/list

Request Parameters

参数 类型 必填 说明
iccid string 否 按 ICCID 筛选
orderNo string 否 按订单号筛选
status string 否 状态：ACTIVE, PENDING, EXPIRED
page integer 否 页码，默认 1
pageSize integer 否 每页数量，默认 20

Cancel Profile

取消/作废指定的 Profile。

POST /profiles/cancel

Request Parameters

参数 类型 必填 说明
iccid string 是 要取消的 Profile ICCID
reason string 否 取消原因

Balance Query

查询集成商账户可用余额，用于确认下单前的账户资金是否充足。

POST /account/balance

Request Parameters

无需额外参数，系统根据请求头中的 AccessCode 识别集成商身份。

Response Example

{
  "code": 0,
  "message": "success",
  "data": {
    "currency": "USD",
    "balance": 125680.00,
    "creditLimit": 50000.00
  }
}

Response Fields

字段 类型 说明
currency string 币种
balance number 可用余额
creditLimit number 信用额度

Top Up

为指定 Profile 购买加油包/充值流量。

POST /profiles/topup

Request Parameters

参数 类型 必填 说明
iccid string 是 Profile ICCID
addonCode string 是 加油包代码
merchantOrderNo string 是 商户订单号（唯一）

Set Webhook

设置 Webhook 回调地址。

POST /webhook/set

Request Parameters

参数 类型 必填 说明
url string 是 Webhook URL（HTTPS）
events array 否 订阅的事件类型，默认全部

Webhook: Order Status

当订单状态发生变化时，系统会向您配置的 Webhook 地址发送通知。适用于异步订购场景，当供应商处理完成后通知订单状态更新。

Event Types

事件类型 说明
order.delivered 订单已发货（供应商异步处理 eSIM 订购成功）
order.failed 订单处理失败
order.cancelled 订单已取消

Payload Example

{
  "event": "order.delivered",
  "timestamp": 1709020800000,
  "data": {
    "orderNo": "ORD-20250227001",
    "merchantOrderNo": "M-20250227-001",
    "status": "DELIVERED",
    "profiles": [
      {
        "iccid": "89852342022273174242",
        "activationCode": "LPA:1$smdp.example.com$...",
        "qrCode": "data:image/png;base64,..."
      }
    ],
    "deliveredAt": "2025-02-27T10:35:00Z"
  }
}

Webhook: Profile Status

当 Profile 状态发生变化时，系统会向您配置的 Webhook 地址发送通知。

Event Types

事件类型 说明
profile.activated Profile 已激活
profile.expired Profile 已过期
profile.data_exhausted 流量已用尽

Payload Example

{
  "event": "profile.activated",
  "timestamp": 1709020800000,
  "data": {
    "iccid": "89852342022273174242",
    "orderNo": "ORD-20250227001",
    "status": "ACTIVE",
    "activatedAt": "2025-02-27T10:30:00Z"
  }
}

Webhook: Balance Alert

当 Profile 流量余额低于阈值时，系统会发送余额预警通知。

Payload Example

{
  "event": "balance.low",
  "timestamp": 1709020800000,
  "data": {
    "iccid": "89852342022273174242",
    "remainingData": "100MB",
    "remainingPercent": 10,
    "threshold": 10
  }
}
