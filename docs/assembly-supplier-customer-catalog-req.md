# 集成商供应侧与对客侧 eSIM 套餐目录设计

## 一、核心功能总结

本文档描述**集成商作为中间供应商**时的 eSIM 套餐与客户侧设计：**供应商的 eSIM 套餐通过供应商 API 同步到管理端**，管理端形成供应商套餐目录并为集成商配置价格与白名单；集成商在供应侧可查看**管理端已为其开通的供应商套餐及报价**，并可基于这些套餐配置**自己的套餐并分配给客户**（对客侧）。客户仅能看到集成商为其配置的 eSIM 套餐与售价，对供应商来源与集成商定价方式无感知，实现**供应透明、对客统一**的中间商模式。**本文档中所述「产品」均指 eSIM 套餐。**

---

## 二、角色与数据边界

| 角色         | 可见范围                                                                 | 不可见 |
|--------------|--------------------------------------------------------------------------|--------|
| **供应商**   | 自己的 eSIM 套餐与报价；向集成商同步的套餐目录与结算数据                 | 集成商的其他供应商、集成商的客户与售价 |
| **集成商**   | 各供应商同步的 eSIM 套餐与报价；自己配置的「我的套餐」、成本与售价、客户分配 | 供应商的其它下游、客户的实际用量明细（按需） |
| **客户**     | 集成商分配给自己的 eSIM 套餐列表、展示名称、规格、**集成商售价**、下单与订单 | 供应商名称、供应商报价、集成商成本、其他客户的套餐或价格 |

---

## 三、业务流程

### （一）供应侧：从供应商套餐到「我的套餐」

1. **管理端**通过**供应商 API** 将供应商的 eSIM 套餐同步到平台，形成管理端「供应商 eSIM 套餐目录」；管理员在「价格与供应商配置」中为各集成商配置可见的供应商/套餐白名单及定价（参见 assembly-1.md）。
2. 集成商在供应侧看到的**供应商套餐及报价**来源于管理端已同步的套餐及为其配置的价格；集成商在**我的套餐配置**中，从这些套餐中选择并创建「我的套餐」，设置对外名称、售价、可见范围（全部客户或指定客户/客户组）。
3. 同一供应商套餐可对应多个「我的套餐」（不同售价或不同客户组），支持差异化定价与选品。

### （二）对客侧：客户只看「我的套餐」

1. 客户通过集成商提供的门户或 API 获取可选购的 eSIM 套餐列表，仅能看到**已分配给该客户（或所属客户组）**的「我的套餐」。
2. 客户看到的内容为：套餐名称、规格（如流量、地区、有效期）、**集成商售价**、下单入口；不展示供应商、成本及集成商定价方式。
3. 客户下单后，订单在集成商侧与平台侧按既有流程处理（与 assembly-req 一致），客户仅关心自己的订单与 eSIM 资产。

### （三）客户与套餐分配

1. 集成商维护**客户**或**客户组**（可选），在「我的套餐」上配置可见范围，或在客户/客户组维度上勾选可见套餐。
2. 分配关系变更后，客户侧可见套餐列表实时或定期更新，无需客户操作。

## 四、供应侧详细需求

### （一）供应商与套餐同步（管理端，参见 assembly-1.md）

供应商 eSIM 套餐的同步在**管理端**完成：管理端维护供应商信息并调用**供应商 API** 将套餐同步到平台，形成「供应商 eSIM 套餐目录」；管理员在「价格与供应商配置」中为集成商配置白名单与定价。集成商侧不直接对接供应商 API，其可见的供应商套餐与报价均来自管理端已同步数据及配置。具体能力包括：

- **供应商列表**：管理端维护供应商名称、编码、API 配置等，支持按供应商、状态筛选；支持手动或按周期触发同步、查看同步日志。
- **同步日志与异常**：按供应商、时间范围查看同步记录，展示成功/失败、失败原因；支持重试，关键失败可告警或纳入待办。
- **供应商 eSIM 套餐目录（管理端只读）**：管理端查看已同步套餐，支持筛选与列表展示；管理员基于此目录为集成商配置价格与白名单。

以下为**集成商端**供应侧需求（集成商所见数据来源于管理端已同步并为其开通的套餐）。

---

### （二）供应商 eSIM 套餐目录（集成商端，只读）

1. **筛选**
   - 支持按**供应商、地区、套餐名称、数据类型、流量规格、有效期、状态**组合筛选。数据来源为管理端已同步且已为该集成商开通（白名单）的供应商套餐。

2. **列表展示字段**（仅供集成商内部使用，不对客户暴露）

| 字段           | 说明                     |
|----------------|--------------------------|
| 供应商         | 来源供应商名称           |
| 供应商套餐ID   | 上游系统 eSIM 套餐唯一标识 |
| 套餐名称       | 供应商侧名称             |
| 地区           | 适用地区                 |
| 流量规格       | 如 1GB/7天               |
| 有效期         | 天数                     |
| 数据类型       | 如 流量型/时长型         |
| 供应商报价     | 成本价/结算价            |
| 货币           | 报价货币                 |
| 最后同步时间   | 该条数据更新时间         |
| 状态           | 正常/停用/同步失败       |

3. **操作**
   - 支持「基于此套餐创建我的套餐」快捷入口，跳转至我的套餐配置并预填供应商套餐关联。
   - 集成商不可在供应商套餐目录中直接编辑或删除；套餐主数据以管理端同步为准，价格与白名单由管理端在「价格与供应商配置」中维护。

---

### （三）我的套餐配置

1. **筛选与列表**
   - 支持按**关联供应商、地区、套餐名称、可见范围（全部/指定客户组）、状态**筛选。
   - 列表展示：我的套餐名称、关联供应商套餐、成本价（来自供应商报价）、售价、可见范围、状态、最近更新时间、操作（编辑、启用/停用、分配客户）。

2. **新增/编辑「我的套餐」表单字段**

| 分类     | 字段           | 说明 |
|----------|----------------|------|
| 关联     | 供应商套餐     | 从「供应商 eSIM 套餐目录」选择一条，必填；选择后可带出规格只读展示 |
| 对内     | 成本价         | 可默认带出供应商报价，可改；仅集成商可见 |
| 对内     | 备注           | 可选，仅集成商可见 |
| 对外     | 对外名称       | 客户看到的套餐名称，必填，可与供应商名称不同 |
| 对外     | 对外描述       | 可选，客户可见 |
| 对外     | 售价           | 客户看到的价格，必填 |
| 对外     | 货币           | 与售价一致 |
| 可见范围 | 可见范围类型   | 全部客户 / 指定客户组 / 指定客户列表 |
| 可见范围 | 客户组/客户    | 当类型为「指定」时，多选客户组或客户 |
| 状态     | 启用/停用      | 停用后客户侧不可见且不可下单 |

3. **批量能力（可选）**
   - 支持批量导入「我的套餐」（模板含：供应商套餐ID、对外名称、售价、可见范围等），支持导出当前配置为 CSV/Excel。

---

### （四）客户与客户组（供应侧）

1. **客户组**
   - 列表：客户组名称、包含客户数、关联「我的套餐」数量、操作（编辑、分配套餐）。
   - 新增/编辑：客户组名称、描述；套餐分配为多选「我的套餐」，保存后该组下客户仅能看到已选套餐。

2. **客户**
   - 列表：客户名称/标识、所属客户组、可见套餐数、状态、操作（编辑、分配套餐）。
   - 可见套餐可继承客户组，并支持客户级覆盖（仅对该客户增加或隐藏部分套餐）。

3. **分配关系维护**
   - 入口一：在「我的套餐」编辑页配置「可见范围」与具体客户组/客户。
   - 入口二：在客户组/客户详情页配置「可见套餐」列表。两处数据一致，可任一侧维护。

---

## 五、对客侧详细需求

### （一）客户可见内容边界

- 客户仅能查看与操作：**已分配给自己的「我的套餐」**、自己的订单、自己的 eSIM 资产、自己的账单（若开放）。
- 客户**不可见**：供应商套餐目录、供应商名称、供应商报价、集成商成本、「我的套餐」的配置与分配管理、其他客户的套餐或价格。

### （二）eSIM 套餐列表（客户视角）

1. **数据来源**
   - 从「我的套餐」中筛选：当前客户或所属客户组在可见范围内，且状态为启用的 eSIM 套餐。

2. **列表展示字段**（客户可见）

| 字段     | 说明 |
|----------|------|
| 套餐名称 | 即「我的套餐」的对外名称 |
| 地区     | 适用地区 |
| 流量规格 | 如 1GB |
| 有效期   | 如 7天 |
| 数据类型 | 如 流量型 |
| 售价     | 集成商配置的售价 |
| 货币     | 售价货币 |

3. **交互**
   - 支持按地区、流量、有效期等筛选（仅就可见套餐做筛选）；支持加入购物车/直接下单、价格查询（Pricing）等，与现有 assembly-req 中「eSIM 套餐」交互一致，但数据源为「我的套餐」且仅限已分配。

### （三）API 与门户一致性

- 客户通过 **API** 拉取 eSIM 套餐列表时，接口返回的即为该客户可见的「我的套餐」子集，字段与对客侧列表一致，不包含供应商、成本等字段。
- 客户通过 **门户** 查看套餐时，展示逻辑与 API 一致，仅展示已分配套餐及集成商售价。

---

## 六、与现有文档的关系

- **assembly-1.md**：管理端「价格与供应商配置」中明确**供应商 eSIM 套餐通过供应商 API 同步到管理端**，管理端维护供应商、同步日志与「供应商 eSIM 套餐目录」，并为集成商配置白名单与定价；本设计中集成商侧可见的供应商套餐及报价即来源于此。
- **assembly-admin-req.md**：平台管理端为集成商配置「供应商白名单、价格体系」等，与 assembly-1 的「价格与供应商配置」一致；本设计中的「供应商」即平台侧为集成商开通的供应来源，套餐主数据由管理端通过供应商 API 同步获得。
- **assembly-req.md**：集成商端的 eSIM 套餐、订单、财务等；本设计将「eSIM 套餐」拆为：
  - **供应侧**：供应商 eSIM 套餐目录（只读，数据来自管理端已同步并下发的套餐）、我的套餐配置、客户/客户组与套餐分配；
  - **对客侧**：客户看到的套餐列表与下单流程（数据来自「我的套餐」+ 分配关系），与 assembly-req 中体验一致，仅数据源与可见性受本设计约束。

## 七、页面/模块清单（供实现对照）

| 类型     | 模块/页面             | 使用角色 | 说明 |
|----------|-----------------------|----------|------|
| 管理端   | 供应商列表与 API 配置 | 管理员   | 维护供应商，通过供应商 API 同步套餐至管理端（见 assembly-1 价格与供应商配置） |
| 管理端   | 同步日志              | 管理员   | 同步记录与异常处理 |
| 管理端   | 供应商 eSIM 套餐目录  | 管理员   | 只读，查看已同步套餐；据此为集成商配置白名单与定价 |
| 供应侧   | 供应商 eSIM 套餐目录  | 集成商   | 只读，查看管理端已为其开通的供应商套餐与报价 |
| 供应侧   | 我的套餐配置          | 集成商   | 新增/编辑/分配「我的套餐」 |
| 供应侧   | 客户组管理            | 集成商   | 客户组及可见套餐 |
| 供应侧   | 客户管理              | 集成商   | 客户及可见套餐（含覆盖） |
| 对客侧   | eSIM 套餐列表（门户） | 客户     | 仅已分配「我的套餐」+ 集成商售价 |
| 对客侧   | eSIM 套餐 API         | 客户系统 | 返回同上，供对接方使用 |

以上清单可直接作为菜单与权限配置、前后端模块划分及接口设计的依据。
