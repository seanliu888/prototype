## 【管理端】

---

## 一、名词定义

以下为本系统中的主要业务概念（简版）：

| 名词 | 定义 |
|------|------|
| 集成商（Integrator） | 接入平台开放 API 的渠道方/商户，基于平台套餐能力对终端用户提供销售与服务。 |
| 供应商（Supplier） | 提供底层 eSIM 套餐与履约能力的上游服务商，套餐基础信息由其 API 同步到平台。 |
| 套餐（Package） | 可被销售和订购的 eSIM 数据产品，通常包含覆盖范围、流量规格、有效期、计费方式等。 |
| 加油包（Top-up Package） | 绑定在已购套餐或 Profile 上的增量流量产品，用于追加流量。 |
| 成本报价 | 平台维护的供应商侧成本价格基线，主要用于内部核算与后续定价。 |
| 指导报价 | 平台级模板报价，用于为一类集成商提供默认/标准销售价格。 |
| 特殊报价 | 针对单个集成商的专属价格配置，可覆盖指导报价。 |
| 报价覆盖 | 当同一套餐同时存在指导报价与特殊报价时，以特殊报价为准的规则。 |
| 计费方式 | 套餐的计价口径，当前主要包括「套餐计费」与「每GB计费」。 |
| Profile（eSIM） | 供应商分配的可激活电子卡档案，关联 ICCID、激活状态、用量与有效期。 |
| 订单（Order） | 集成商通过 API 提交的购买请求记录，包含订单号、金额、状态与履约结果。 |
| 交易记录（Transaction） | 账户资金变动流水，包含充值、消费、调账、退款等类型。 |
| 结算单（Settlement） | 按账期汇总应收应付数据形成的对账单据，用于确认、收款/付款与销账。 |
| AccessCode / SecretKey | 集成商调用 API 的身份凭证与签名密钥。 |
| Webhook | 平台向集成商异步推送状态变更事件的回调机制。 |




## 二、需求概要描述

### （一）集成商管理（Integrators）

1. **集成商多条件筛选**
   - 支持通过**集成商名称/编号、状态（正常/停用/待审核）、创建时间区间、联系人邮箱/手机** 组合筛选。

2. **集成商列表展示**
   - 以列表形式展示集成商核心信息：名称、编号、状态、联系人、账户余额、创建时间，并提供「查看详情」「编辑」「停用/启用」等操作入口。

3. **新增/编辑集成商**
   - **新增**：支持录入企业名称、联系人姓名、邮箱、手机、结算方式（预付/后付）、备注等；创建后自动生成平台账号（或由管理员指定初始密码/发送激活链接）。
   - **编辑**：支持修改集成商基本信息、状态（启用/停用）、结算方式及备注。

4. **集成商详情**
   - 进入单个集成商详情页，可查看其完整档案。

---

### （二）价格与供应商配置（Pricing & Suppliers）

1. **供应商与套餐同步（管理端）**
   - **供应商维护**：支持维护供应商信息（名称、编码、API 配置等），支持按供应商名称、状态筛选与列表展示。
   - **套餐同步**：支持通过**供应商 API** 将供应商的 eSIM 套餐同步到管理端；支持手动触发同步或配置同步周期，同步结果在管理端形成「供应商 eSIM 套餐目录」。
   - **同步日志与异常**：支持按供应商、时间范围查看同步记录，展示同步时间、成功/失败、失败原因；支持对失败项重试，关键失败可纳入告警或待办。
   - **成本报价**：管理端在「报价管理」下可查看已同步的套餐列表（即供应商套餐目录），支持按供应商、地区、套餐名称等筛选；套餐基础信息由供应商 API 同步，**仅报价与计费方式**可在此编辑或导入。

2. **指导报价（价格模板）**
   - 支持维护平台级价格模板（如默认指导价、银牌/金牌合作伙伴），为不同级别的集成商设置标准报价方案；新接入集成商可绑定默认模板，未单独定价的套餐可使用「兜底指导报价」模板。
   - 模板下可配置各套餐的指导价、建议零售价；支持新建模板（可从空白或复制现有模板）、编辑、启用/停用、查看使用该模板为兜底报价的集成商列表。

3. **特殊报价（集成商专属定价）**
   - 支持按集成商维度，从成本报价（已同步套餐）中为其配置套餐售价、成本价（可选）、建议零售价等；支持批量导入/导出某集成商的价格表。
   - 支持为不同集成商配置不同套餐可见范围；集成商侧可见的套餐与报价来源于成本报价及此处配置；与指导报价配合：可先绑定指导报价模板，再对部分套餐做特殊报价覆盖。

4. **配置筛选与展示**
   - 支持按**集成商、地区、套餐名称、数据类型** 筛选已配置的套餐与价格，以列表形式展示集成商、套餐、售价、成本、生效时间等，并提供「编辑」「批量调整」入口。

---
### （三）财务与结算（Billing & Settlement）

1. **交易记录与调账**
   - 支持按**集成商、交易类型、时间区间、交易 ID** 筛选交易记录；列表展示交易 ID、集成商、金额、类型、余额快照、时间、备注。
   - 管理员可执行**人工调账**（如补偿、扣减、备注）。

---

### （四）订单与用量（Orders & Usage）

1. **订单筛选**
   - 支持通过**订单号、集成商、订单状态、创建时间区间、金额区间** 筛选全平台或指定集成商的订单，提供「重置」「收起筛选」功能。

2. **订单列表与详情**
   - 以列表形式展示订单号、所属集成商、eSIM 数量、金额、状态、创建时间等；支持点击进入订单详情，查看关联的 eSIM 列表及状态，便于客诉与对账。

---

### （五）API 管理（Developer & API）

1. **密钥与回调监管**
   - 管理员可查看或修改各集成商的 API 密钥状态、Webhooks 配置；在风控或客诉场景下支持「强制重置密钥」或「临时禁用 API」。

2. **开放接口功能**
   - 查询套餐：请求提供所有可用数据包的列表。可选择按国家或地区过滤。
   - 订阅套餐：单独或批量订购配置套餐。
   - 退订套餐：取消未激活的esim套餐（需供应商支持）。
   - 查询订单：支持分页查询过往订单，获取订单详情，获取esim状态、激活码等信息。

3. **开发文档**
   - 提供「API 版本与文档」说明。

## 三、场景需求描述

### 场景一：新集成商接入

**场景说明**：平台运营人员完成新集成商的入驻配置，使其能够通过 API 订购 eSIM 套餐。

**必有流程**：

1. **创建集成商账号**
   - 运营人员进入「集成商管理」→「新增集成商」
   - 填写企业名称、联系人信息（姓名、邮箱、手机）
   - 选择结算方式（预付/后付），设置信用额度（可选）
   - 保存后系统自动生成集成商编号（如 INT-2025001）

2. **配置 API 密钥**
   - 系统自动为新集成商生成 AccessCode 和 SecretKey
   - 运营人员在「API 管理」页面可查看密钥（脱敏显示）
   - 如需配置 IP 白名单或 Webhook 地址，点击「编辑」进行设置

3. **配置套餐定价**
   - 进入「报价管理」→「特殊报价」→「批量导入」
   - 第一步：选择目标集成商
   - 第二步：从成本报价（套餐目录）中筛选并勾选要开放给该集成商的套餐
   - 第三步：选择定价方式
     - 方式一：应用定价模板（如报价 + 30% 利润率）
     - 方式二：上传 CSV 文件手动设置售价
   - 确认导入后，该集成商即可通过 API 查询并订购这些套餐

4. **账户充值（预付模式）**
   - 集成商完成线下汇款后，运营人员进入「财务与结算」
   - 点击「充值 / 调账」，选择集成商，输入充值金额
   - 确认后账户余额更新，集成商可开始下单

---

### 场景二：供应商套餐同步与上架

**场景说明**：平台对接新供应商或更新现有供应商的套餐目录，并将套餐配置给集成商销售。

**必有流程**：

1. **添加/维护供应商**
   - 运营人员进入「供应商管理」
   - 如为新供应商，需先在系统中创建供应商记录（名称、编码）
   - 供应商 API 配置由技术人员在后台完成

2. **触发套餐同步**
   - 进入供应商详情页 →「同步」Tab
   - 点击「同步」按钮手动触发，或等待系统定时自动同步
   - 同步完成后，套餐数据写入「套餐目录」

3. **查看同步结果**
   - 在「同步」Tab 查看同步日志：同步时间、结果、数量、失败原因
   - 如有失败记录，点击「重试」重新同步
   - 进入「套餐目录」验证套餐是否正确同步（套餐名称、流量、有效期、报价等）

4. **为集成商配置定价**
   - 进入「报价管理」→「特殊报价」→「批量导入」
   - 筛选新同步的套餐，勾选后为指定集成商设置售价
   - 或在特殊报价列表中单独「编辑」某条定价，调整售价和建议零售价
   - 配置完成后，集成商 API 即可查询到新套餐

---

### 场景三：集成商 API 下单

**场景说明**：集成商通过 API 为终端用户订购 eSIM 套餐，平台完成订单处理和 eSIM 分配。

**必有流程**：

1. **集成商查询套餐**
   - 集成商调用 `Get All Data Packages` 接口
   - 可按地区/国家筛选，获取可订购的套餐列表（仅返回已为该集成商配置定价的套餐）
   - 返回信息包括：套餐Code、名称、流量、有效期、售价等

2. **集成商提交订单**
   - 集成商调用 `Order Profiles` 接口
   - 传入套餐Code、订购数量、商户订单号
   - 系统校验：套餐是否可售、账户余额是否充足（预付模式）
   - 校验通过后，系统向供应商下单获取 eSIM

3. **订单处理与扣费**
   - 系统生成平台订单号（如 ORD-20250227001）
   - 从集成商账户扣除订单金额，生成消费类型的交易记录
   - 订单状态为「已支付」，等待供应商处理

4. **供应商发货**
   - 系统向供应商提交订单请求
   - 供应商处理完成后返回 eSIM 信息
   - 订单状态更新为「已发货」，记录供应商交易号

5. **返回 eSIM 信息**
   - 接口返回订单号及 eSIM 列表
   - 每张 eSIM 包含：ICCID、激活码、二维码
   - 集成商将激活信息提供给终端用户

6. **管理端查看订单**
   - 运营人员可在「套餐订单」列表查看该订单
   - 点击「详情」查看关联的 eSIM 列表及状态
   - 在「eSIM管理」可按 ICCID、订单号等筛选查看

---

### 场景四：eSIM 激活与用量监控

**场景说明**：终端用户激活 eSIM 后，平台实时同步状态和用量信息，支持运营人员监控和客服查询。

**必有流程**：

1. **用户激活 eSIM**
   - 终端用户扫描二维码或输入激活码，在设备上安装 eSIM Profile
   - 供应商侧检测到激活后，通过 Webhook 通知平台
   - 平台接收 `profile.activated` 事件，更新 eSIM 状态为「已激活」

2. **用量数据同步**
   - 平台定期从供应商获取 eSIM 用量数据
   - 更新「eSIM管理」列表中的已用流量、剩余流量、剩余时间

3. **集成商查询用量**
   - 集成商调用 `Query All Allocated Profiles` 接口，可按 ICCID 筛选
   - 返回该 eSIM 的状态、流量使用情况等信息

4. **运营人员监控**
   - 运营人员在「eSIM管理」可按状态筛选（已激活/待激活/已过期）
   - 查看统计信息：合计数量、可用数量
   - 点击「详情」查看单张 eSIM 的完整信息和已购加油包

---

### 场景五：加油包购买

**场景说明**：终端用户流量不足时，集成商通过 API 为其购买加油包补充流量。

**必有流程**：

1. **查询可用加油包**
   - 集成商调用 `Get All Data Packages` 或查看套餐详情
   - 获取该套餐关联的加油包列表（加油包Code、流量、售价）

2. **提交加油包订单**
   - 集成商调用 `Top Up` 接口
   - 传入 ICCID、加油包Code、商户订单号
   - 系统校验：eSIM 状态是否有效、账户余额是否充足

3. **订单处理**
   - 系统向供应商提交加油包订单
   - 扣除集成商账户余额，生成交易记录
   - 加油包流量叠加到该 eSIM

4. **管理端查看**
   - 运营人员在「eSIM详情」页可查看该 eSIM 的已购加油包列表
   - 显示加油包名称、流量、购买时间、状态（使用中/未使用/已用完）

---

### 场景六：账户充值与调账

**场景说明**：运营人员为集成商账户充值或进行人工调账（补偿/扣减）。

**必有流程**：

1. **充值操作**
   - 集成商完成线下汇款（如对公转账）
   - 运营人员进入「财务与结算」，点击「充值 / 调账」
   - 选择集成商，操作类型选「充值」，输入金额和备注
   - 确认后账户余额增加，生成充值类型的交易记录

2. **调账操作（扣减）**
   - 如需纠错或扣款，操作类型选「扣减」
   - 输入扣减金额和原因备注
   - 确认后账户余额减少，生成调账类型的交易记录

3. **查看交易记录**
   - 在交易记录列表可按集成商、交易类型、时间筛选
   - 每条记录显示：交易ID、金额（正/负）、类型、余额快照、时间、备注
   - 便于财务对账和审计追溯

---

### 场景七：账期结算与对账

**场景说明**：财务按结算周期（如周结/月结）对集成商进行应收结算，并完成与供应商的成本对账与结算。

**必有流程**：

1. **生成结算单**
   - 财务选择结算周期（如 2026-03-01 至 2026-03-31）和结算对象（指定集成商/全部）
   - 系统汇总周期内交易与订单数据，生成结算单（结算单号、周期、应收金额、已收金额、未收金额）
   - 对已作废、已退款、冲正订单按规则剔除或净额计算

2. **账单核对**
   - 财务在结算单中查看明细：订单金额、退款金额、调账金额、期初/期末余额
   - 支持按订单号、交易ID追溯到原始记录
   - 对异常差异（金额不一致、缺失单据）标记并记录处理备注

3. **确认与出账**
   - 核对无误后将结算单状态从「草稿」提交为「待确认」
   - 集成商确认后状态变更为「已确认」；如有异议可退回并重新出账
   - 支持导出结算单（CSV/PDF）用于外部财务归档

4. **收款与销账**
   - 财务登记收款（线下转账/其他渠道），录入收款时间与流水号
   - 系统将结算单状态更新为「部分支付」或「已结清」
   - 自动生成对应交易记录并更新应收余额

5. **供应商成本结算**
   - 按供应商维度汇总周期内成本，生成供应商应付结算单
   - 对账通过后登记付款信息，更新为「已支付」
   - 保留供应商结算明细，用于毛利分析与审计追溯

---

### 场景八：API 风控与密钥管理

**场景说明**：当发现集成商 API 异常调用或安全风险时，运营人员进行风控处置。

**必有流程**：

1. **发现异常**
   - 监控发现某集成商 API 调用异常（如频繁失败、疑似被盗用）
   - 或收到集成商反馈密钥泄露

2. **临时禁用 API**
   - 运营人员进入「API 管理」
   - 找到该集成商，点击「禁用 API」
   - API 状态变为「已禁用」，该集成商所有 API 调用将被拒绝

3. **重置密钥**
   - 点击「重置密钥」，系统生成新的 SecretKey
   - 旧密钥立即失效
   - 通知集成商更新密钥配置

4. **配置 IP 白名单**
   - 点击「编辑」，在 IP 白名单中添加集成商服务器 IP
   - 配置后仅允许白名单内的 IP 调用 API
   - 增强安全性，防止密钥被滥用

5. **恢复 API 访问**
   - 确认风险解除后，点击「启用 API」
   - API 状态恢复为「启用」，集成商可正常调用

## 四、需求详细描述

### 首页概览

#### 1. 首页概览页

**页面入口**：左侧导航「首页概览」

**左侧导航结构**：
- 首页概览
- **供应商**：供应商管理
- **集成商**：集成商管理、报价管理（折叠子项：成本报价、指导报价、特殊报价）

#### 3. API 版本与文档页

**页面入口**：API 管理页「API 版本与文档」按钮

**页面结构**：
- 顶部导航栏：返回按钮、版本徽章（v1.0）
- 左侧文档目录：可折叠的 API 分类导航
- 右侧文档内容：API 详细说明

**开放 API 功能列表**：
| 接口 | 功能说明 |
|------|----------|
| Get All Data Packages | 获取所有可用数据套餐列表，支持按地区/国家筛选 |
| Order Profiles | 订购 eSIM Profile，支持批量订购 |
| Query All Allocated Profiles | 分页查询已分配的 Profile 列表 |
| Cancel Profile | 取消/作废指定的 Profile |
| Balance Query | 查询集成商账户可用余额，用于确认下单前资金是否充足 |
| Top Up | 为指定 Profile 购买加油包/充值流量 |
| Set Webhook | 设置 Webhook 回调地址 |
| Supported Regions | 获取支持的地区和国家列表 |

**Webhook 事件**：
| 事件 | 说明 |
|------|------|
| Profile Status | Profile 状态变更通知（激活、过期、流量用尽） |
| Balance Alert | 流量余额低于阈值时的预警通知 |
