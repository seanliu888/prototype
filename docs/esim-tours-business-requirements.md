# eSIM TOURS Business 功能需求文档

## 一、需求背景

面向可登录 eSIM TOURS Business 平台的企业成员，承载 eSIM 采购、资产管理、订单管理、API 对接、团队用户、额度账务和账单等核心业务模块。

## 二、角色定义

eSIM TOURS Business 平台主要包含以下 4 类角色：

| 角色 | 定义 | 平台访问范围 |
| --- | --- | --- |
| 管理员（Admin） | 拥有 eSIM TOURS Business 平台全部功能权限，可管理业务、财务、用户和 API 相关能力 | 全量访问 |
| 运营（Operations） | 拥有核心运营功能权限，可进行 eSIM 采购、订单处理、eSIM 管理、用户分配和用量监控 | 运营访问 |
| 财务（Finance） | 拥有财务相关功能权限，可查看订单、额度和账单信息 | 财务访问 |
| 员工（Employee） | 可登录 eSIM TOURS Business 平台，访问 eSIM 商店、eSIM 管理和订单菜单，用于提交购买申请、查看个人 eSIM 和相关订单 | 受限访问 |

说明：

1. 管理员可在添加用户时分配角色，也可在 `Users` 页面后续编辑用户角色。
2. 员工仅展示 `eSIMs store`、`eSIMs`、`Orders` 菜单，不应允许访问其他后台路由。
3. 角色权限需要同时作用于菜单展示、页面路由和后端接口，避免仅前端隐藏菜单造成越权访问。

## 四、使用流程

### （一）平台开户与角色配置

1. 企业账号开通后，管理员登录 eSIM TOURS Business 平台。
2. 管理员进入 `Users` 添加企业成员，并为成员分配管理员、运营、财务或员工角色。
3. 管理员和运营可进行业务管理；财务可查看订单、余额和账单；员工仅可使用 `eSIMs store`、`eSIMs`、`Orders`。
4. 员工账号创建后，可在 `eSIMs` 查看被分配或共享的 eSIM，也可进入 `eSIMs store` 选择套餐并发起购买申请。

### （二）成员邀请与加入流程

1. 管理员进入 `Users`，填写成员姓名、邮箱和角色，向成员发送邀请。
2. 系统生成邀请记录，并将成员状态标记为待邀请或待加入。
3. 被邀请成员通过邮件或邀请链接完成账号注册、登录或加入组织。
4. 成员加入成功后，系统更新组织加入状态，并按分配角色授予对应访问权限。
5. 邀请过期、未收到邀请或成员未加入时，管理员可重新发送邀请或取消邀请。

### （三）员工购买申请流程

1. 员工进入 `eSIMs store`，按目的地、流量规格、有效期、价格等条件筛选套餐。
2. 员工选择目标套餐后提交购买申请，不直接生成订单或扣减余额。
3. 系统记录申请人、申请套餐、使用场景、预计使用时间、申请说明等信息。
4. 申请提交后进入后台待审核状态，并通知管理员或运营处理。
5. 员工可在 `Orders` 查看申请状态，包括待审核、已通过、已驳回、采购中、已完成等。

### （四）后台审核与采购流程

1. 管理员或运营进入后台查看员工提交的购买申请。
2. 审核人员根据员工需求、套餐可用性、余额情况和企业规则决定通过或驳回申请。
3. 审核通过后，系统按员工申请中选择的套餐自动发起采购。
4. 自动采购完成后系统生成订单，并将对应 eSIM 关联到员工。
5. 审核驳回时需填写原因，员工可在 `Orders` 查看驳回说明。

### （五）订单与 eSIM 管理流程

1. 管理员、运营和财务可进入 `Orders` 查看页面下单、API 下单和员工申请触发的订单；员工可进入 `Orders` 查看本人购买申请和本人 eSIM 关联订单。
2. 管理员和运营可进入 `eSIMs` 查看已购买、已分配、已激活的 eSIM 资产；员工可进入 `eSIMs` 查看本人 eSIM。
3. 管理员或运营可查看 eSIM 详情、用量、有效期，并执行重新发送、分配和导出等操作。
4. 员工通过 `eSIMs` 查看已分配 eSIM 的二维码、激活信息、有效期和用量状态。

### （六）API 对接流程

1. 管理员进入 `API integration` 查看或生成 Access Code、Secret Key。
2. 管理员配置 Webhook 回调地址，并参考 API 文档完成系统对接。
3. API 下单产生的订单统一进入 `Orders`，通过订单来源区分。
4. API 来源订单详情中展示商户订单号、API 请求流水号、回调状态和失败原因等排查信息。

### （七）余额与账单流程

1. 管理员和财务进入 `Credits` 查看账户余额、充值记录、扣款记录和调整记录。
2. 当余额不足时，管理员或财务按企业付款流程进行充值或线下处理。
3. 管理员和财务进入 `Billing` 查看账单、发票、付款记录和对账明细。
4. 财务可下载账单或发票，用于内部财务核对。

## 五、详细功能需求

### （一）首页（Home）

1. 展示合作伙伴账户的核心业务指标，包括 eSIM 采购数量、激活数量、订单数量、账户余额或可用额度等。
2. 支持展示待处理事项，例如待邀请用户、待审核员工购买申请、失败订单、余额不足、API 异常、账单待支付等。
3. 提供快捷入口跳转至常用模块，例如 eSIM 商店、订单、账单、API 集成。
4. 支持按时间维度查看业务概览，如今日、本周、本月、近 30 天。
5. 财务进入首页时应突出财务相关指标，运营进入首页时应突出订单、eSIM 和用量相关指标。

### （二）eSIM 商店（eSIMs store）

1. 支持浏览可购买的 eSIM 套餐列表。
2. 支持按国家/地区、套餐名称、流量规格、有效期、价格、网络类型等条件筛选。
3. 展示套餐关键信息，包括套餐名称、覆盖地区、流量、有效期、售价、网络运营商和库存状态。
4. 支持管理员和运营选择套餐并进入采购流程。
5. 支持员工选择套餐并提交购买申请，申请通过后由系统自动采购。
6. 支持批量采购或加入购物车，具体模式按平台采购流程确定。
7. 对不可购买、库存不足或无权限购买的套餐进行明确提示。
8. 财务不可访问 eSIM 商店；套餐浏览、采购和员工购买申请仅面向管理员、运营和员工开放。

### （三）eSIM 管理（eSIMs）

1. 展示企业已购买、已分配或已激活的 eSIM 资产列表。
2. 支持按 ICCID、订单号、套餐名称、国家/地区、eSIM 状态、激活状态等条件查询。
3. 展示 eSIM 的生命周期状态，包括未使用、已分配、已安装、已激活、已过期、已取消等。
4. 支持查看 eSIM 详情，包括二维码、SM-DP+ 地址、激活码、流量使用量、有效期和关联订单。
5. 支持查看 eSIM 用量监控信息，包括已用流量、剩余流量、激活时间、到期时间和异常使用提示。
6. 支持管理员和运营执行下载二维码、重新发送、分配用户、导出等操作。
7. 员工可访问 `eSIMs`，但仅能查看被分配或共享给自己的 eSIM。
8. 对敏感信息展示和导出需进行权限控制和操作日志记录。

### （四）订单（Orders）

1. 展示平台内订单列表，包含页面下单、API 下单和员工申请审核通过后产生的订单。
2. 支持按订单号、采购日期、订单状态、套餐名称、金额区间、订单来源等条件筛选。
3. 展示订单核心字段，包括订单号、创建时间、订单来源、eSIM 数量、订单金额、支付状态和订单状态。
4. 订单来源用于区分订单创建渠道，建议至少包含页面下单、API 下单、员工申请三类。
5. 支持查看订单详情，包含订单明细、套餐信息、关联 eSIM、付款信息、来源信息和操作记录。
6. API 来源订单详情中应展示必要的接口排查信息，例如商户订单号、API 请求流水号、回调状态、失败原因等。
7. 员工申请来源订单应展示申请人、申请说明、审核人、审核时间和审核结果。
8. 订单详情应支持查看订单内 eSIM 的使用概览，辅助运营进行用量监控和问题排查。
9. 支持订单数据分页、排序和导出。
10. 对失败、取消、部分完成等异常订单提供清晰状态说明。
11. 财务可查看订单及财务相关字段；运营可查看并处理运营相关订单状态；管理员拥有完整订单权限；员工仅可查看本人购买申请和本人 eSIM 关联订单。

### （五）API 集成（API integration）

1. 展示企业 API 对接配置，包括 Access Code、Secret Key、回调地址、白名单和接口状态。
2. 支持生成、重置和停用 API 密钥。
3. 支持配置 Webhook 回调地址，并提供测试回调能力。
4. 支持查看 API 文档入口、接口版本说明和环境信息。
5. 对密钥仅支持脱敏展示，复制或重置密钥需进行二次确认。
6. API 配置变更需记录操作人、操作时间和变更内容。
7. API 集成配置属于高风险能力，默认仅管理员可访问和操作。

### （六）用户（Users）

1. 展示企业成员列表，并在页面标题或列表区域展示当前成员总数。
2. 支持按姓名或邮箱搜索用户。
3. 支持按用户状态筛选，例如 Active、Inactive、Pending 等。
4. 展示用户字段，包括姓名、状态、邮箱、组织加入状态、角色和操作入口。
5. 支持批量选择用户，并展示已选用户数量。
6. 支持邀请用户、添加用户、编辑用户、停用用户、删除用户、重新发送邀请等操作。
7. 支持管理员、运营、财务、员工四类角色分配。
8. 支持组织状态展示，例如 Not invited、Joined。
9. 支持邀请状态管理，包括待发送、已发送、已加入、已过期、已取消等状态。
10. 支持管理员重新发送邀请、取消邀请和查看邀请发送时间。
11. 管理员可新增用户并分配任意角色，也可后续编辑角色。
12. 运营可执行用户分配相关操作，但是否允许创建、停用或修改财务/管理员角色需进一步确认。
13. 员工角色用户仅可访问 `eSIMs store`、`eSIMs`、`Orders`，用于提交购买申请、查看本人 eSIM 和本人相关订单。

### （七）余额（Credits）

1. 展示账户当前可用额度、冻结金额、信用额度或预付余额。
2. 展示充值、扣款、退款、调整等余额变动记录。
3. 支持按交易类型、时间范围、金额、交易 ID 等条件查询。
4. 支持查看单笔交易详情，包括关联订单、变动前后余额和备注。
5. 支持导出余额流水，用于财务核对。
6. 余额不足或额度异常时应提供明显提示和处理入口。
7. Credits 属于财务功能，默认仅管理员和财务可访问。

### （八）账单（Billing）

1. 展示账单、发票、付款记录和结算状态。
2. 支持按账期、付款状态、账单类型、金额区间等条件筛选。
3. 展示账单字段，包括账单号、账期、金额、币种、状态、生成时间和到期时间。
4. 支持下载账单、下载发票、查看付款记录和对账明细。
5. 支持展示线上支付或线下转账信息，具体支付方式按平台能力确定。
6. 财务敏感操作需进行权限控制和操作日志记录。
7. Billing 属于财务功能，默认仅管理员和财务可访问。
